# EcoPulse — Carbon Footprint Tracker

> A personal carbon footprint web app that helps you **track, understand, and reduce** your environmental impact through simple lifestyle inputs and AI-powered recommendations.

Built as a single-container web application: **Python / FastAPI** backend with a **React + TypeScript** frontend, powered by **Google Gemini (Vertex AI)** for personalized advice and **Firestore** for anonymous tracking, deployed to **Google Cloud Run**.

---

## Overview

EcoPulse lets you estimate your annual carbon footprint across four lifestyle categories — transport, home energy, diet, and consumption — then delivers personalized, quantified actions to help you reduce it.

| Phase | What happens |
|---|---|
| **Track** | Enter everyday habits → get an annual footprint broken down by category with comparisons to global averages and sustainable targets. |
| **Understand** | See which areas of your life contribute the most emissions, ranked and visualized. |
| **Reduce** | Receive 2–4 tailored recommendations with estimated annual savings, generated by Gemini AI with a rule-based fallback. |

Save snapshots over time (anonymously via a device ID) to monitor your progress.

## Architecture

```
Browser (React + TS via Vite)            Cloud Run (single container)
  • Accessible UI + bar chart ──HTTP──► FastAPI
  • Anonymous device ID (localStorage)    ├─ POST /api/calculate  — emission engine
                                          ├─ POST /api/insights   — Gemini → fallback
                                          ├─ POST /api/entries    — save snapshot
                                          ├─ GET  /api/entries/{id} — history
                                          └─ GET  / (+ assets)    — serves SPA
                                              │
                                              ├─ Vertex AI (Gemini) via ADC
                                              └─ Firestore (Native) via ADC
```

A single container serves both API and static files — one origin, no CORS in production. Google Cloud authentication uses Application Default Credentials (no secrets in the repo).

## API endpoints

| Method | Path | Purpose |
|---|---|---|
| `POST` | `/api/calculate` | Footprint breakdown for given lifestyle inputs |
| `POST` | `/api/insights` | Personalized reduction advice (Gemini or rules) |
| `POST` | `/api/entries` | Save a footprint snapshot |
| `GET` | `/api/entries/{device_id}` | Retrieve device history |
| `GET` | `/api/health` | Liveness check |

## Local development

**Backend** (Python 3.10+):

```bash
cd backend
python -m venv .venv && .venv\Scripts\Activate
pip install -r requirements-dev.txt
USE_GEMINI=false USE_FIRESTORE=false uvicorn app.main:app --reload
```

**Frontend** (Node 20+):

```bash
cd frontend
npm install
npm run dev        # proxies /api to localhost:8000
```

**Docker** (whole stack):

```bash
docker build -t ecopulse .
docker run -p 8080:8080 -e USE_GEMINI=false -e USE_FIRESTORE=false ecopulse
```

## Design decisions

- **Emission factors** sourced from UK DEFRA 2023, US EPA, and IPCC / Our World in Data — every constant in `backend/app/carbon/factors.py` cites its source.
- **Graceful AI degradation**: Gemini generates personalized advice when available; if unreachable, a deterministic rule engine ensures users always get quantified guidance. Responses are tagged `source: "gemini" | "rules" | "cache"`.
- **Anonymous by design**: no login or PII. A random device ID in localStorage keys all history.
- **Accessibility-first**: semantic HTML, labelled controls, skip link, keyboard navigation, AA-contrast theme, prefers-reduced-motion support, and automated axe checks per component.
- **Defense in depth**: input validation at three levels (browser max attributes, Pydantic bounds on the API, body-size middleware).
